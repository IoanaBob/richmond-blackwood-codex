# RB Process Maintenance

Use this when updating RB SOPs or process maps.

## Workflow

1. Read `processes/index.md`.
2. Add new steps to the most specific process file.
3. Keep private client examples out of broad process docs.
4. Link to client folders for exceptions or evidence.
5. Back up general SOP changes to the RB Internal Knowledge Base.
6. Update `sources/import-log.md` and `memory/handoff.md`.

## Fact Status

New process facts are provisional until reviewed.
